## Core Constraints — Follow These Exactly

1. **No build tools.** No npm, no webpack, no Vite, no bundler of any kind. The project must remain deployable by dropping files on a static host.
2. **No external CSS frameworks.** All styling is vanilla CSS with custom properties inside `index.html`. Do not introduce Tailwind or Bootstrap.
3. **One file for the app.** All HTML, CSS, and JavaScript live in `index.html`. Do not split into separate `.css` or `.js` files unless the user explicitly requests it.
4. **Use `var` in JavaScript.** The existing codebase uses `var` throughout. Do not refactor to `const`/`let` unless specifically asked.
5. **No pip dependencies.** `fetch_tornadoes.py` uses only Python standard library. Keep it that way.
6. **Never edit `tornadoes.geojson`.** It is generated by the GitHub Actions pipeline.

## Data Pipeline

### Source
NOAA Storm Prediction Center CSV — public domain, no API key needed.  
URL pattern: `https://www.spc.noaa.gov/wcm/data/1950-{YEAR}_actual_tornadoes.csv`

### CSV Column Mapping → GeoJSON Properties
| CSV column | GeoJSON property | Notes |
|------------|-----------------|-------|
| `yr`, `mo`, `dy` | `date` (YYYY-MM-DD) | |
| `time` | `time` (HH:MM) | HHMM integer → formatted |
| `st` or `stf` | `state` | Falls back to FIPS lookup |
| `mag` | `ef` | Clamped 0-5; -9 → 0 |
| `inj` | `injuries` | |
| `fat` | `fat` | fatalities |
| `wid` | `width_yd` | yards |
| `len` | `len_mi` | miles |
| `slat`, `slon` | `geometry.coordinates[0]` | start point |
| `elat`, `elon` | `geometry.coordinates[1]` | end point; stubbed if 0,0 |

### Important Edge Cases in the Script
- **Missing end coords (`0,0`):** Stub a 0.01° east offset so Leaflet renders it
- **Positive longitudes:** CONUS longitudes should be negative; script corrects if positive
- **Unknown rating (`mag = -9`):** Treat as EF0
- **No start coord:** Skip the record entirely

## Leaflet-Specific Notes

- **Tile layer:** CartoDB Dark Matter (`https://{s}.basemaps.cartocdn.com/dark_all/{z}/{x}/{y}{r}.png`)
- **GeoJSON layer:** Created with `L.geoJSON(data, { style, onEachFeature })`, stored in `tornadoLayer`
- **Popup:** `L.popup({ maxWidth: 300 })` — styled via `.leaflet-popup-content-wrapper` CSS overrides
- **Style reset:** Call `tornadoLayer.resetStyle(layer)` to restore a layer after hover/deselect
- **Layer refresh:** On data re-fetch, the old `tornadoLayer` is removed with `map.removeLayer()` before creating a new one
- **Fit bounds:** Called after layer creation with `.pad(0.1)` margin

## GitHub Actions Workflow Details

**File:** `.github/workflows/fetch-tornado-data.yml`

```yaml
on:
  schedule:
    - cron: "0 6 * * 1"   # Every Monday 06:00 UTC
  workflow_dispatch:
    inputs:
      start_year: { default: "2000" }

permissions:
  contents: write           # Required to commit tornadoes.geojson back to repo
```

The workflow:
1. Checks out the repo
2. Sets up Python 3.11 (no pip cache — no requirements.txt exists)
3. Optionally patches `START_YEAR` constant if manually dispatched with a year
4. Runs `python scripts/fetch_tornadoes.py` from the repo root
5. Commits `tornadoes.geojson` only if it changed (idempotent)

**Common failure modes:**
- `cache: pip` in `setup-python` fails without `requirements.txt` → don't add it
- Workflow permissions must be set to "Read and write" at the repo level
- NOAA CSV URL must be updated when NOAA releases the next year's data

## Testing Checklist

When making changes to `index.html`, verify:
- [ ] Filters (month, day, EF) correctly show/hide tracks
- [ ] Clicking a track opens both the popup and the sidebar detail panel
- [ ] Reset button restores all tracks and clears the detail panel
- [ ] Refresh (⟳) button clears localStorage and re-fetches data
- [ ] Loading overlay disappears after data renders
- [ ] "Updated" stat pill shows the correct date from `metadata.generated`
- [ ] Mobile: sidebar toggles correctly at ≤800px viewport
- [ ] No visible tracks → "No results" notice appears

When modifying `scripts/fetch_tornadoes.py`, verify:
- [ ] Script runs with `python scripts/fetch_tornadoes.py` from repo root
- [ ] Output file is valid JSON parseable by `json.load()`
- [ ] All features have `LineString` geometry with ≥2 coordinate pairs
- [ ] `metadata.record_count` matches `len(features)`
- [ ] No external imports (only stdlib)

## Tone & Style for Code Changes

- **HTML:** Semantic elements, ARIA labels on interactive elements, `id` attributes match JS selectors exactly
- **CSS:** Use custom properties from `:root`. Prefer flexbox. Micro-animations via `transition` and `animation`. Dark glassmorphism aesthetic.
- **JavaScript:** Vanilla ES5-compatible. No arrow functions (use `function`). No template literals (use string concatenation). No destructuring. Matches existing style throughout.
- **Python:** PEP 8. Docstrings on module and functions. Constants in UPPER_SNAKE_CASE. Graceful error handling with informative print statements.
- **YAML:** 2-space indent, comments explaining every non-obvious step.
