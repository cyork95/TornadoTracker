## Project Overview

**TornadoTracker** is a single-page web application that maps historical US tornado tracks using Leaflet.js. It is a fully static site (no backend, no build step) deployed via GitHub Pages at `tornadotracker.yorkdevelops.com`.

**Owner:** cyork95 / yorkdevelops.com  
**Repo:** https://github.com/cyork95/TornadoTracker  
**Live site:** https://tornadotracker.yorkdevelops.com

---

## Architecture

```
index.html          ← Entire app: HTML + embedded CSS + embedded JS
tornadoes.geojson   ← Auto-generated by GitHub Actions (never hand-edit)
scripts/
  fetch_tornadoes.py          ← Python 3 stdlib only, no pip deps
.github/workflows/
  fetch-tornado-data.yml      ← Weekly cron, manual dispatch
CNAME                         ← tornadotracker.yorkdevelops.com
README.md
```

**No framework. No bundler. No npm. No build step.** Everything is vanilla HTML/CSS/JS inside a single `index.html`. Keep it that way unless explicitly asked to change the stack.

---

## Key Design Decisions

| Decision | Rationale |
|----------|-----------|
| Single `index.html` | Zero-dependency deploy — drop on any static host |
| `var` instead of `const/let` in JS | Broad browser compatibility; consistent with existing code style |
| `localStorage` cache (6-hr TTL) | Avoids re-fetching 7 MB GeoJSON on every page load |
| CartoDB Dark Matter tiles | Free, no API key, visually consistent with dark theme |
| NOAA SPC CSV (not an API) | No rate limits, no auth, public domain data |
| Python stdlib only in script | Zero pip install step in GitHub Actions |
| Fallback to mock data | App still works when `tornadoes.geojson` isn't deployed yet |

## CSS Architecture

All CSS lives in a single `<style>` block inside `index.html`.

**Design token conventions (CSS custom properties):**
- `--bg-darkest`, `--bg-dark`, `--bg-card`, `--bg-card-hover` — background layers
- `--border-subtle`, `--border-glow` — border states (idle vs. hover/focus)
- `--accent-blue` (`#4f90ff`), `--accent-cyan` (`#00d4ff`) — interactive accents
- `--text-primary`, `--text-muted`, `--text-dim` — text hierarchy
- `--ef0` through `--ef5` — EF scale color palette (yellow → deep purple)
- `--radius-sm` (8px), `--radius-md` (14px), `--radius-lg` (20px)
- `--transition: all 0.22s cubic-bezier(0.4, 0, 0.2, 1)` — standard easing

**Do not** introduce Tailwind, Bootstrap, or any external CSS framework.

## JavaScript Architecture

All JS lives in a single `<script>` block at the bottom of `index.html`.

### Global variables (all `var`)
```js
CONFIG          // map settings, track weights/opacities
EF_SCALE        // array of {id, color, label, desc, glow}
TORNADO_GEOJSON // built-in mock data (fallback only)
GEOJSON_URL     // './tornadoes.geojson'
CACHE_KEY / CACHE_TS_KEY / CACHE_TTL_MS  // localStorage cache config
tornadoLayer    // Leaflet GeoJSON layer (null until data loads)
allFeatures     // full feature array
activeFeature   // currently selected Leaflet layer
filterMonth / filterDay / filterEF  // current filter state
```

### Key functions
| Function | Purpose |
|----------|---------|
| `efColor(rating)` | Returns hex color for EF 0-5 |
| `efInfo(rating)` | Returns full EF metadata object |
| `buildLegend()` | Populates `#ef-legend` grid |
| `initTornadoLayer(geojsonData)` | Creates Leaflet GeoJSON layer, adds to map |
| `featureStyle(feature)` | Returns Leaflet style object per feature |
| `bindFeatureEvents(feature, layer)` | Attaches hover/click handlers |
| `onTrackClick(e)` | Highlights track, opens popup, shows sidebar detail |
| `showMapPopup(props, latlng)` | Renders Leaflet popup HTML |
| `showDetailPanel(props)` | Populates `#detail-card` in sidebar |
| `applyFilters()` | Filters visible layers by month/day/EF |
| `updateStats()` | Refreshes header stat pills |
| `fetchGeoJSON(forceRefresh)` | Cache-or-fetch entry point |
| `renderData(data, source)` | Tears down old layer, re-initializes, hides loader |
| `loadFromCache()` | Reads `localStorage`, checks TTL |
| `saveToCache(data)` | Writes GeoJSON + timestamp to `localStorage` |
| `clearCache()` | Removes both localStorage keys |
| `setLoadingText(msg, sub)` | Updates loading overlay text |
| `showUpdatedTimestamp(isoStr)` | Updates `#stat-updated` pill |
| `hideLoader()` | Adds `.hidden` class to `#loading-overlay` |
| `bindUI()` | Wires all filter controls and buttons |

### Data flow on page load
```
DOMContentLoaded
  → buildLegend()
  → bindUI()
  → setTimeout 400ms → fetchGeoJSON(false)
      → loadFromCache() → hit? → renderData(cached, 'cache')
                        → miss? → fetch('./tornadoes.geojson')
                                    → ok  → saveToCache() → renderData(data, 'network')
                                    → err → renderData(TORNADO_GEOJSON, 'fallback')
```

## GeoJSON Schema

Each feature must have:
```json
{
  "type": "Feature",
  "geometry": { "type": "LineString", "coordinates": [[lon, lat], [lon, lat]] },
  "properties": {
    "ef":       0,           // integer 0-5
    "date":     "2023-05-03", // YYYY-MM-DD
    "time":     "14:22",      // HH:MM
    "state":    "KS",         // 2-letter abbreviation
    "location": "...",        // display string
    "width_yd": 25,           // integer, yards
    "len_mi":   3.2,          // float, miles
    "injuries": 0,            // integer
    "fat":      0             // integer, fatalities
  }
}
```

The root object also has a `metadata` block:
```json
{
  "metadata": {
    "generated":    "2026-06-21T23:56:25.034488+00:00",
    "source":       "https://...",
    "start_year":   2000,
    "record_count": 31298
  }
}
```

## Python Script (`scripts/fetch_tornadoes.py`)

- **Python 3.6+, zero external deps** (uses `csv`, `json`, `urllib`, `datetime`, `os`, `io` only)
- Tries multiple NOAA SPC CSV URLs in order (they change filenames each year)
- Filters to `START_YEAR` (default 2000) — change this constant to adjust coverage
- Creates LineString geometry from `slat/slon` → `elat/elon`; stubs a tiny offset when end coords are `0,0`
- Corrects positive longitudes (SPC sometimes omits the minus sign for CONUS)
- Output is compact JSON (no indentation) to keep file size minimal

## GitHub Actions Workflow

- **Schedule:** Every Monday 06:00 UTC (`0 6 * * 1`)
- **Manual dispatch:** Supports optional `start_year` input parameter
- **Permissions:** Needs `contents: write` (set at repo level)
- **No secrets needed** — NOAA data is fully public
- Commits only if `tornadoes.geojson` actually changed (idempotent)

## Gotchas & Known Issues

1. **NOAA changes the CSV filename yearly** — the script tries 2024, 2023, 2022 in order
2. **`cache: pip` breaks `setup-python`** without a `requirements.txt` — we don't use it (no pip deps)
3. **Large real dataset** — 31,298 features at 7.3 MB; on slow connections the load overlay shows a URL
4. **Longitude correction** — SPC CSV sometimes has positive values for CONUS; script negates them
5. **`0,0` end coordinates** — tornado end points are often missing; script stubs a tiny line segment
6. **localStorage quota** — 7.3 MB is near browser limits; `saveToCache` silently skips on quota error
7. **Leaflet popup cleanup** — `popupclose` event resets `activeFeature` and calls `tornadoLayer.resetStyle()`

## What NOT to Do

- ❌ Do not add npm, webpack, Vite, or any build toolchain
- ❌ Do not add external CSS frameworks (Tailwind, Bootstrap, etc.)
- ❌ Do not hand-edit `tornadoes.geojson` — it is auto-generated
- ❌ Do not add `cache: pip` to the GitHub Actions workflow without also adding `requirements.txt`
- ❌ Do not change `var` to `const/let` in bulk — respect the existing style
- ❌ Do not add `<script src="...">` tags for libraries without asking first
- ❌ Do not split `index.html` into separate files without an explicit request

## Testing Locally

```bash
# Serve the directory (fetch() requires HTTP, not file://)
python -m http.server 8080

# To test with real data first:
python scripts/fetch_tornadoes.py
python -m http.server 8080
# Open http://localhost:8080
```

To test the fallback (no GeoJSON): rename/remove `tornadoes.geojson` and reload.  
To test cache behavior: open DevTools → Application → Local Storage → clear keys prefixed with `tornadotracker_`.
