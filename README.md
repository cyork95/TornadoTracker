**[→ View Live Site](https://tornadotracker.yorkdevelops.com)**

---

## ✨ Features

- 🗺️ **Full-screen interactive map** — dark-themed CartoDB basemap centered on the continental US
- 🎨 **EF-scale color coding** — tracks visually graded from EF0 (yellow) through EF5 (deep purple)
- 🔍 **Filter by Month & Day** — compare historical patterns on any date across all years
- 📋 **Click-to-inspect** — popup + sidebar panel showing EF rating, date/time, path width/length, injuries, and fatalities
- ⚡ **Live stats bar** — visible tracks, total records, fatality count update in real time
- 💾 **Smart caching** — GeoJSON is cached in `localStorage` for 6 hours; hits the server only when stale
- 🔄 **One-click refresh** — ⟳ button in the header forces an instant data reload
- 📱 **Responsive** — sidebar collapses on mobile with a floating toggle button

## 🏗️ How It Works

```
NOAA SPC Public CSV (1950–present)
         │
         │  Every Monday 06:00 UTC
         ▼
  GitHub Actions Workflow
  scripts/fetch_tornadoes.py
         │  Parses ~30k tornado records
         │  Converts to GeoJSON LineStrings
         │  Commits tornadoes.geojson → repo
         ▼
  tornadoes.geojson  (in this repo)
         │
         │  GitHub Pages serves it automatically
         ▼
  tornadotracker.yorkdevelops.com
         │
         │  Browser fetches on page load
         │  localStorage cache (6-hr TTL)
         ▼
    Leaflet.js renders tracks on map  ✅
```

No backend, no API keys, no database — just a static site and a scheduled GitHub Action.

## 🚀 Deployment

This site is deployed via **GitHub Pages** from the `master` branch root.

### GitHub Actions — Automated Data Pipeline

The workflow [`.github/workflows/fetch-tornado-data.yml`](.github/workflows/fetch-tornado-data.yml) runs automatically every Monday at 06:00 UTC. It:

1. Downloads the latest NOAA SPC tornado CSV
2. Converts it to GeoJSON (tornado records from **year 2000 onward** by default)
3. Commits `tornadoes.geojson` back to the repo
4. GitHub Pages immediately serves the updated file

You can also trigger it manually from the **Actions** tab at any time.

#### Changing the Year Range

Edit `START_YEAR` in [`scripts/fetch_tornadoes.py`](scripts/fetch_tornadoes.py):

```python
START_YEAR = 2000   # Change to 0 for all data back to 1950 (~70k records)
```

Or pass it as a parameter when triggering manually via **Actions → Fetch Tornado Data → Run workflow**.

## 🌐 Custom Domain Setup (DNS)

The site uses the custom domain `tornadotracker.yorkdevelops.com`. To replicate this on your own domain, add a **CNAME record** at your DNS provider:

| Type | Host/Name | Value | TTL |
|------|-----------|-------|-----|
| `CNAME` | `tornadotracker` | `cyork95.github.io` | 3600 |

Then update the [`CNAME`](CNAME) file in this repo with your domain.

> **Note:** DNS propagation can take up to 24 hours. GitHub Pages will also provision a free TLS certificate automatically once the DNS record resolves.

## 🛠️ Running Locally

No build step required — just open `index.html` in a browser.

To test with real data locally, run the fetch script first:

```bash
# Python 3.6+ required, no extra packages needed
python scripts/fetch_tornadoes.py

# Then serve the directory (required for fetch() to work)
python -m http.server 8080
# Open http://localhost:8080
```

## 📊 Data Source

All tornado data is sourced from the **NOAA Storm Prediction Center (SPC) Severe Weather Database**:

- **URL:** https://www.spc.noaa.gov/wcm/#data
- **Format:** CSV, updated annually (previous year added each spring)
- **Coverage:** Every recorded US tornado from 1950 to present
- **License:** Public domain (US Government data)

Each tornado record includes: date, time, state, F/EF rating, injuries, fatalities, path start/end coordinates, path length (miles), and path width (yards).

## ⚙️ GeoJSON Schema

Each feature in `tornadoes.geojson` follows this structure:

```json
{
  "type": "Feature",
  "geometry": {
    "type": "LineString",
    "coordinates": [[-97.14, 35.42], [-96.30, 35.80]]
  },
  "properties": {
    "ef":       2,
    "date":     "2023-04-19",
    "time":     "20:05",
    "state":    "OK",
    "location": "OK Tornado #12",
    "width_yd": 300,
    "len_mi":   14.5,
    "injuries": 18,
    "fat":      1
  }
}
```

The file also includes a `metadata` block with `generated` (ISO timestamp), `source` (URL), `start_year`, and `record_count`.
